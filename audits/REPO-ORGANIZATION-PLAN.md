# StreetJS Repository Organization Plan

> **Status:** Recommendations only. **Nothing in this plan is executed automatically.**
> No file is moved, renamed, or deleted, and no code is modified by this document.
> Every move below is proposed as a manual `git mv` so commit history is preserved.
> Historical/internal files are retained — they are relocated within the repo (or to a
> separate repo), not removed.

## How this plan was produced

Every recommendation is based on the live repository state, verified with:

```bash
find . -maxdepth 1 -type f | sort        # 60+ root files
find . -maxdepth 1 -type d | sort        # root directories
git ls-files | wc -l                     # 1747 tracked files
git ls-files --error-unmatch <file>      # tracked vs untracked
git check-ignore <file>                  # gitignored?
```

Verified facts that shaped the recommendations:

| Item | Tracked? | Gitignored? |
|------|----------|-------------|
| `certification-report.json` | untracked | **ignored** |
| `RELEASE-CERTIFICATION.md` | untracked | **ignored** |
| `street-signing.key.pem` / `.pub.pem` | untracked | **ignored** |
| `.env` | untracked | **ignored** |
| `sbom.json` | **TRACKED** | not ignored |
| `release-inputs.json` | **TRACKED** | not ignored |
| `BingSiteAuth.xml` | **TRACKED** | not ignored |
| `googledf528d4f2b039b20.html` | **TRACKED** | not ignored |
| `app-htmx/` `app-next/` `app-none/` `app-react/` | **TRACKED** (36/34/21/33 files) | not ignored |

Existing strong structure: `packages/` (49 packages), `examples/` (13), `rfcs/` (4),
`docs/`, `scripts/`, `benchmarks/`, `.github/` (40 workflows), plus `deploy/`, `demos/`,

## 1. Comparison to mature framework roots

Mature framework monorepos keep their root lean. The root is the project's front door —
it holds discovery/legal/governance metadata, the workspace manifest, tooling config, and
pointers into well-named subdirectories. Strategy, audits, and generated artifacts do not
live there.

What the reference roots actually keep (observed pattern across the five):

| Project | Root holds (essentially only) |
|---------|-------------------------------|
| `vercel/next.js` | `readme.md`, `license.md`, `contributing.md`, `code-of-conduct.md`, `security.md`, `package.json`, `pnpm-lock.yaml`, `pnpm-workspace.yaml`, tooling config, `.github/`, `packages/`, `docs/`, `examples/`, `scripts/`, `bench/`, `test/` |
| `nestjs/nest` | `readme.md`, `LICENSE`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `package.json`, `lerna.json`, lockfile, tsconfig, `.github/`, `packages/`, `sample/`, `integration/`, `scripts/`, `benchmarks/` |
| `expressjs/express` | `Readme.md`, `LICENSE`, `Contributing.md`, `Code-of-conduct.md`, `Security.md`, `History.md`, `package.json`, `index.js`, `lib/`, `test/`, `examples/`, `benchmarks/` |
| `nuxt/framework` (nuxt) | `README.md`, `LICENSE`, `package.json`, `pnpm-workspace.yaml`, lockfile, config, `.github/`, `packages/`, `docs/`, `examples/`, `playground/`, `scripts/`, `test/` |
| `laravel/framework` | `README.md`, `LICENSE.md`, `CONTRIBUTING.md`, `composer.json`, `phpunit.xml`, `src/`, `tests/`, `bin/` |

**Common denominator at root:** `README`, `LICENSE`, `SECURITY`, `CONTRIBUTING`,
`CODE_OF_CONDUCT`, `GOVERNANCE`/`MAINTAINERS` (where present), `CHANGELOG`/`History`,
manifest + lockfile + workspace file, tooling dotfiles, `.github/`, `packages/`, `docs/`,
`examples/`, `scripts/`, `benchmarks/`. **And little else.**

**Where StreetJS diverges:** the framework-grade layout already exists
(`packages/`, `docs/`, `examples/`, `rfcs/`, `scripts/`, `benchmarks/`, `.github/`), but
the root is buried under ~30 internal planning/strategy/roadmap docs, ~13 completed
audit/report docs, 6 scattered `docker-compose.*.yml` files, 4 generated scaffold apps
(`app-*`), website SEO verification files, and a couple of tracked generated artifacts.
That is the entire gap between StreetJS and the reference roots.

## 4. Organization Score: **58 / 100**

| Dimension | Score | Why |
|-----------|------:|-----|
| Package / monorepo layout | 18/20 | `packages/` with 49 packages is framework-grade |
| Docs / examples / rfcs | 16/20 | `docs/`, `examples/` (13), `rfcs/` all present and healthy |
| CI / governance metadata | 14/15 | 40 workflows, CODEOWNERS, GOVERNANCE, MAINTAINERS, SECURITY, CoC |
| Root cleanliness | 4/25 | ~30 internal planning + ~13 audit `.md` files crowd the root |
| Deployment layout | 3/10 | 6 scattered `docker-compose.*` at root + a separate `deploy/` |
| Artifact / secret hygiene | 3/10 | `sbom.json` & `release-inputs.json` tracked; SEO files tracked; scaffold `app-*` tracked at root |
| **Total** | **58/100** | |

**Justification.** The bones of a mature framework repo are already in place — a real
package workspace, dedicated docs, examples, RFCs, scripts, benchmarks, and a deep CI +
governance setup. That foundation alone is most of the way to framework-grade. The score
is held back almost entirely by **root noise**: roughly 30 internal strategy/roadmap docs
and a dozen completed audits living at root, six `docker-compose.*.yml` files plus a
parallel `deploy/` tree, four generated scaffold apps (`app-*`), website SEO files, and a
couple of generated artifacts (`sbom.json`, `release-inputs.json`) that should be CI
outputs. None of this requires code changes — it is purely an organization problem, which
is why the ceiling is high once §2 relocations land.

## 5. Phased migration plan

All moves use `git mv` (preserves history). **No step deletes or auto-removes anything**;
removals are explicit, opt-in decisions noted as such. Run from repo root.

### Immediate — within 7 days (low risk, high signal)

Clears the root with pure relocations. No behavior change.

```bash
# Internal strategy/roadmap docs -> plans/
mkdir -p plans
git mv STREETJS-EXPANSION-MASTERPLAN.md COMMUNITY-GROWTH-PLAN.md COMMUNITY-ROADMAP.md \
       CONTENT-DRAFTS.md CONTENT-ROADMAP.md DEMO-INFRA-PLAN.md CLI-EVOLUTION.md \
       WEBSITE-EVOLUTION.md PLUGIN-MARKETPLACE-PLAN.md SHOWCASE-GALLERY-PLAN.md \
       SHOWCASE-ROADMAP.md STARTER-CATALOG-PLAN.md STARTERS-ROADMAP.md \
       SECURITY-ROADMAP.md TRUST-CENTER-PLAN.md ADOPTION-ASSETS.md \
       STREETJS-HTMX-PLAN.md PHASE-18-EXECUTION-PLAN.md plans/

# Completed audits/reports -> audits/
mkdir -p audits
git mv SECURITY-AUDIT-2026.md THREAT-MODEL-2026.md THREAT-MODEL-UPDATE.md \
       PAYMENTS-SECURITY-REVIEW.md PLUGIN-SIGNING-REVIEW.md ECOSYSTEM-PLUGINS-AUDIT.md \
       ENTERPRISE-READINESS.md LANGUAGE-STATS-AUDIT.md SECURITY-SCORECARD.md \
       MARZPAY-INTEGRATION-REPORT.md STREET_WEBSITE_ENTERPRISE_AUDIT.md \
       PHASE-18-AUDIT.md SHOWCASE-AUDIT.md audits/

# User-facing docs -> docs/
git mv PLUGIN-MARKETPLACE.md docs/
git mv GOOD-FIRST-ISSUES.md docs/        # or .github/

# Root smoke script -> scripts/
git mv .sqlite-smoke.mjs scripts/sqlite-smoke.mjs   # then update any reference in package.json/CI
```

Also in this window (artifact hygiene, opt-in):
- Stop tracking generated artifacts and add to `.gitignore`:
  ```bash
  git rm --cached sbom.json release-inputs.json
  printf "\nsbom.json\nrelease-inputs.json\n" >> .gitignore   # review before committing
  ```
  Then ensure CI emits them as release artifacts.
- Add `plans/README.md` and `audits/README.md` explaining each folder's purpose, and link
  the standing security deliverables from `SECURITY.md`.

> A short note on the deliverable audit docs: the security/threat-model deliverables
> (`SECURITY-AUDIT-2026.md`, `THREAT-MODEL-2026.md`, etc.) are intentionally kept — they
> simply live under `audits/` and are referenced from `SECURITY.md`, so they remain
> discoverable without crowding the root.

### Medium — within 30 days (consolidation, touches CI/build)

```bash
# Consolidate deployments
mkdir -p deployments/compose deployments/docker
git mv docker-compose.yml docker-compose.kafka.yml docker-compose.rabbitmq.yml \
       docker-compose.search.yml docker-compose.storage.yml docker-compose.test-db.yml \
       deployments/compose/
git mv Dockerfile deployments/docker/Dockerfile        # optional; keep .dockerignore at root

# Fold existing deploy/ into deployments/
git mv deploy/k8s deployments/kubernetes
git mv deploy/helm deployments/helm
mkdir -p deployments/examples
git mv deploy/aws-ecs deploy/cloud-run deploy/cloudflare deploy/vercel deployments/examples/
git mv deploy/README.md deployments/README.md
```
Then update references: `docker compose -f deployments/compose/...` in scripts/CI, any
`docker build -f deployments/docker/Dockerfile .`, and the relevant `.github/workflows/*`
(`kafka-integration.yml`, `rabbitmq-integration.yml`, `mongodb-integration.yml`,
`deploy-verify.yml`, etc.). Verify each affected workflow locally/in a branch before merge.

Scaffold apps decision (opt-in, per directory):
```bash
# If curated examples:
git mv app-htmx examples/scaffold-htmx
git mv app-next examples/scaffold-next
git mv app-none examples/scaffold-none
git mv app-react examples/scaffold-react
# If CLI golden output: git mv app-* test/fixtures/scaffold/<name>
# If throwaway generator output: consider `git rm -r` + regenerate in CI (explicit choice)
```

### Long — within 90 days (separation & policy)

- **Move SEO files to the website repo**, then remove here once they live there:
  ```bash
  git rm BingSiteAuth.xml googledf528d4f2b039b20.html   # after they exist in the site repo
  ```
- **Internal strategy split:** evaluate moving `plans/` (and optionally
  `STREET_WEBSITE_ENTERPRISE_AUDIT.md`) into a private `streetjs-internal` repo if any of
  it is strategically sensitive. Keep public-relevant roadmaps (e.g. a trimmed public
  roadmap) in `docs/`.
- **Signing keys policy:** confirm `street-signing.key.pem` stays gitignored forever
  (private key). If a public verification key must ship, commit it deliberately under the
  owning package (e.g. `packages/<signing>/keys/street-signing.pub.pem`) rather than root.
- **Add a CI guard** (a workflow or a `scripts/check-root.mjs`) that fails if new
  unexpected files appear at repo root, locking in the cleaned structure.
- **Decide on `CLAUDE.md`:** keep as local agent context (add to `.gitignore`) or formalize
  under `docs/` if it is meant to be shared.

### Summary of what changes vs. what is preserved

- **Preserved as-is:** `packages/`, `docs/`, `examples/`, `rfcs/`, `scripts/`,
  `benchmarks/`, `.github/`, all root metadata (README/LICENSE/SECURITY/GOVERNANCE/etc.),
  and all already-correct gitignored secrets/artifacts.
- **Relocated (history kept):** internal docs → `plans/`, audits → `audits/`, feature docs
  → `docs/`, compose + `deploy/` → `deployments/`, smoke script → `scripts/`.
- **Opt-in / explicit decisions only:** untrack `sbom.json` & `release-inputs.json`,
  relocate or remove `app-*`, remove SEO files after they move to the website repo.
- **Nothing is auto-removed by this document.**
