# Interactive DevTools

`@streetjs/devtools` is the Street Framework **Interactive Developer Experience**:
a single, self-contained browser bundle that lets you explore and exercise a
running Street application without writing throwaway scripts. It delivers four
tools:

- **Playground** — route, middleware, and plugin testing plus an OpenAPI viewer.
- **Route Explorer** — a visual route tree where each registered route shows its
  HTTP method and path.
- **Dependency Graph Visualizer** — the application's module dependencies drawn
  as nodes and edges.
- **API Inspector** — issue a request and inspect the response status, headers,
  and body. On failure it shows an error indication and keeps your input.

The tools are sourced from the framework's own pure data builders in
`streetjs` — `buildRouteTree`, `buildDependencyGraph`,
`inspectorSuccess`/`inspectorFailure`, and `openApiToHtml` — so the framework and
the tools agree, byte-for-byte, on the data model.

## Security model: token-gated and read-only

The DevTools surface is a privileged window into a running app, so its access
model is **declared and enforced** — it is not just documentation.

- **Authentication (token-gated).** Every request the tools make against the
  inspected app requires a bearer access token. The raw token is never stored;
  the server-side `DevtoolsAuthGate` keeps only its SHA-256 hash and compares in
  constant time. A missing, empty, or wrong token is rejected with `401`.
- **Authorization (read-only).** Even an authenticated caller may only issue
  **safe** methods — `GET`, `HEAD`, `OPTIONS`. Any state-changing method
  (`POST`, `PUT`, `PATCH`, `DELETE`, …) is rejected with `403`. The tools cannot
  mutate the inspected app.

Enforcement lives in `DevtoolsAuthGate` (the authoritative server-side gate) and
is mirrored in the browser client: no request leaves the page until a token is
entered, mutating operations are visibly disabled, and the token is held in
memory only.

```ts
import { DevtoolsAuthGate } from '@streetjs/devtools';

const gate = DevtoolsAuthGate.fromToken(process.env.DEVTOOLS_TOKEN!);

// Mount in front of the inspected app: fail-closed on token and method.
const decision = gate.authorizeHeader(req.headers.authorization, req.method);
if (!decision.allowed) {
  res.statusCode = decision.status; // 401 or 403
  res.end(decision.reason);
  return;
}
```

## Generating the bundle

The static bundle embedded here is produced by the package's CLI and written
into `docs/devtools/index.html`:

```bash
# from packages/devtools
npm run build:docs        # builds the package, then emits docs/devtools/index.html
# or directly:
npx streetjs-devtools --out ../../docs/devtools/index.html
```

For a live application, assemble the dataset programmatically and render the
bundle yourself:

```ts
import { buildDevtoolsData, renderDevtoolsBundle } from '@streetjs/devtools';

const data = buildDevtoolsData(app, {
  title: 'My API DevTools',
  baseUrl: 'https://api.example.com',
  dependencyEntry: 'src/main.ts',
});
const html = renderDevtoolsBundle(data);
```
