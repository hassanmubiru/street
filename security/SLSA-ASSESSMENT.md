# StreetJS SLSA Assessment

> Assessment against [SLSA v1.0](https://slsa.dev) Build track. Evidence-based.

## Current level: **SLSA Build L2** (with several L3 controls already in place)

| Requirement | Track/Level | Status | Evidence |
|---|---|---|---|
| Scripted/automated build | L1 | ✅ | `ci-cd.yml`, `publish-plugins.yml` (no manual build) |
| Provenance exists | L1 | ✅ | npm `--provenance` (in-toto/SLSA provenance via npm) |
| Build runs on hosted CI | L2 | ✅ | GitHub-hosted runners |
| Provenance is **signed/authenticated** | L2 | ✅ | npm provenance signs via OIDC `id-token: write`; cosign release signing |
| Provenance generated by the build service (not user) | L3 | ◑ | npm provenance is GitHub-OIDC-backed (qualifies); plugin Ed25519 signing uses a CI secret key (key-based, not yet hardware/keyless) |
| Isolated, ephemeral build environment | L3 | ◑ | GitHub-hosted ephemeral runners ✅; no explicit hardening/attestation of isolation |
| Hermetic / reproducible build | (beyond L3) | ⬜ | Not asserted; `npm ci` from lockfile is deterministic but not hermetic |

## Missing requirements for full L3
1. **Two-person review on `main`** (branch protection) — required for trusted provenance.
2. **Keyless or KMS-backed signing** for plugin manifests (currently a CI-secret Ed25519 key) — removes long-lived key risk.
3. **Provenance verification gate at install/consume time** (verify the SLSA provenance + manifest signature before trusting).

## Recommended improvements
- Enable branch protection (gates trusted provenance) — see `BRANCH-PROTECTION-REVIEW.md`.
- Migrate plugin signing to Sigstore keyless (OIDC) or a KMS/HSM key (also addresses the leaked-key history item).
- Add a CI job that verifies published tarball provenance + manifest signature (extend `verify-signing-anchor`).
- Document the build as L2-attested in the Trust Center; target L3 after the above.

## Required CI changes (summary)
- Branch protection (platform) + required reviews.
- Keyless signing workflow (replace `STREET_PLUGIN_SIGNING_KEY` secret with OIDC).
- `verify-provenance` job consuming `npm audit signatures` / cosign verify.
