# plans/

Internal strategy, roadmap, and planning documents. These are **internal** —
they describe direction, marketing, and growth, not shipped behaviour. Consider
moving anything strategically sensitive to a private repository.

Contents include expansion/master plans, community & content roadmaps, showcase
and starter catalog plans, CLI/website evolution, the security roadmap, and
phase execution plans. See the repository Governance Charter
([`../governance/CHARTER.md`](../governance/CHARTER.md) §10) for the documentation policy.
